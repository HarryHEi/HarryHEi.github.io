    简介

    git本地仓库由三个东西构成:

    • 工作目录
    • 暂存区
    • HEAD

      git init

      切换到工作目录，使用git init初始化仓库

      git config

      信息配置

    git help

    获得帮助，后面可以添加参数

    git add

    将文件添加到暂存区，可以使用

    添加所有文件

    git mv

    使用mv指令更改文件名或者移动文件

    git rm

    删除文件

    git commit -m “some message”

    将改动提交到HEAD，并标注一段信息

    加上a将会更新暂存区内容并commit

    git log

    查看commit id

    查看特定author的提交信息

    git status

    查看工作区状态
    如果工作区有改动，会和暂存区对比并显示

    git diff

    显示工作区和暂存区的不同点

    加入参数staged，能够显示与HEAD之间的不同点

    git checkout

    将某个文件回滚到暂存区的状态

    新分支整合到主分支后面

    给某个版本打上标签

    将某个文件回滚到某个commit版本

    git reset HEAD

    将某个文件重置到HEAD里面的版本，注意，后面的版本会被删除

    git remote add

    连接远程仓库

    git remote -v

    查看连接的远程仓库

    git push :

    本地HEAD推送到远程仓库
